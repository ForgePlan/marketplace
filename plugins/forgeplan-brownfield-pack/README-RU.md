# forgeplan-brownfield-pack

> Превратить унаследованную кодовую базу плюс накопившуюся документацию в структурированный граф forgeplan — с явной разметкой степени уверенности для каждого бизнес-утверждения.

Пакет для маркетплейса Claude Code, занимающийся **извлечением знаний из brownfield-проектов**: берёт старый код, хранилища Obsidian, наборы ADR в формате MADR, выгрузки из Confluence и т.п., и производит граф артефактов forgeplan — PRD, RFC, ADR, Evidence — где каждое утверждение связано с источником и помечено степенью уверенности.

> [!WARNING]
> Требуется CLI `forgeplan` v0.25+ и среда исполнения playbook-ов (EPIC-007 / PRD-065 в исходном forgeplan). Содержимое скиллов поставляется уже сейчас; часть оркестрации playbook-ов зависит от возможностей forgeplan CLI которые ещё выкатываются.

## Быстрый старт

```bash
/plugin install forgeplan-brownfield-pack@ForgePlan-marketplace
/reload-plugins
```

## Двухуровневое извлечение в двух словах

Методология разделяет:

- **Уровень 1 — Factum**: что код реально делает, доказуемо чтением. Уверенность 100%, проверяется повторным `grep`. Примеры: значения ENUM, ветки условий, форма возвращаемого значения.
- **Уровень 2 — Intent**: почему бизнес выбрал такую реализацию. Уверенность переменная — каждое утверждение помечено: `подтверждено` ✅ / `уверенно выведено` 🟢 / `выведено` 🟡 / `предположение` 🟠 / `неизвестно` ⬜.

Это разделение защищает от самой частой ошибки: подачи гипотез как фактов.

## Порядок работы

```
forgeplan init                                  # если ещё не настроен
# Запускаем brownfield-разведку (Discover Agent в agents/discover/)

# Дальше связка скиллов извлечения (типичный порядок):
ubiquitous-language     → docs/glossary.md
use-case-miner          → docs/use-cases/*.md
intent-inferrer         → docs/hypotheses/*.md
invariant-detector      → docs/invariants/*.md
causal-linker           → перекрёстные ссылки в графе знаний
hypothesis-triangulator → решает какие гипотезы выживают после проверки
interview-packager      → пакует открытые вопросы для владельца предметной области
scenario-writer         → docs/scenarios/*.md (Given/When/Then)
kg-curator              → приводит граф в порядок
canonical-reproducer    → воспроизводит ключевой поток
reproducibility-validator → подтверждает что воспроизведение соответствует реальности
rag-packager            → пакует всё для RAG-выдачи

# На выходе: граф forgeplan с PRD, RFC, ADR, Evidence
```

## Когда брать этот пакет

- Достался унаследованный код без внутренней документации.
- Накопилась куча старых документов (Obsidian, MADR, Confluence) и хочется уложить их в структурированный граф forgeplan.
- Нужно извлечь бизнес-логику перед крупной переработкой.
- Нужен RAG-набор данных по бизнес-знаниям извлечённый из существующего кода.

## Когда НЕ брать этот пакет

- Новый проект с нуля — используй [`fpl-skills`](../fpl-skills/README-RU.md), команды `/fpl-init` и `/forge-cycle`.
- Код уже хорошо задокументирован — извлечение лишнее, просто приведи в порядок что есть.
- Нужна только статическая документация (Obsidian, Confluence) — этот пакет производит именно граф forgeplan.
