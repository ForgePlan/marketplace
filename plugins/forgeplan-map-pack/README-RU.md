# forgeplan-map-pack

> Сканирует любой forgeplan-проект через оркестрованный конвейер из 8 агентов и выдаёт один провалидированный `.forgeplan/map/map.json` — данные, по которым composed-map-вид в forgeplan-web рисует интерактивную зонированную карту проекта.

Пакет для маркетплейса Claude Code, закрывающий разрыв между уже отгруженным рендерером composed-map в forgeplan-web (фаза P0) и реальными данными проекта: до этого пакета сканера не существовало вовсе, и рендерер рисовал только рукописную тестовую заглушку. Этот пакет — генерирующая половина: полностью оркестрованный, детерминированно-гейтованный конвейер, а не короткий скрипт.

> [!WARNING]
> Требует `forgeplan` CLI v0.25+ (playbook-runtime + read-only MCP-поверхности `forgeplan_graph`/`list`/`get`). Сканирование — **только локальный headless-запуск**: сервер `forgeplan-web` структурно не может запустить `claude` (он шеллит только бинарник `forgeplan`, и только read-only подкоманды), поэтому кнопки «запустить анализ» в веб-интерфейсе нет и не будет. `/map-build` (или эквивалентный headless-вызов `claude -p`) запускаете сами, в целевом репозитории.

## Быстрый старт

```bash
/plugin install forgeplan-map-pack@ForgePlan-marketplace
/reload-plugins
```

Затем, в любом forgeplan-проекте:

```bash
claude -p '/map-build' --add-dir <репо> --allowedTools Read Glob Grep Write Task
```

Это сканирует репозиторий, классифицирует тип проекта, извлекает граф зон/узлов/рёбер и пишет `.forgeplan/map/map.json` со статусом `"proposed"`. Дальше детерминированный guardian-гейт либо переводит его в `"confirmed"`, либо чётко сообщает, что именно блокирует.

## Что внутри

### 8 агентов (`agents/`)

Полный оркестрованный конвейер — детали в [ARCHITECTURE.md](ARCHITECTURE.md). Коротко:

| Агент | Стадия | Роль |
|---|---|---|
| `map-orchestrator` | дирижёр | Диспетчеризует все стадии, проверяет 4 гейта, сам ничего не пишет |
| `code-scanner` | SCAN | Дерево исходников, манифесты, точки входа → свой scratch-файл |
| `forgeplan-scanner` | SCAN | Граф артефактов `.forgeplan/`, read-only MCP → свой scratch-файл |
| `docs-scanner` | SCAN | README/доки → факты RU-нарратива, только реальная проза, никогда не выдумывается |
| `zone-extractor` | EXTRACT | Сливает 3 скана в зоны/узлы/мега-узлы, минтит content-hash id |
| `edge-verifier` | VERIFY | Классифицирует рёбра, grep-подтверждает кодовые зависимости, отбрасывает неподтверждённое |
| `map-emitter` | EMIT | Единственный автор содержимого `map.json`, атомарная запись, `status: "proposed"` |
| `map-guardian` | VALIDATE | Запускает детерминированный скрипт-гейт; только его `exit 0` — путь к `"confirmed"` |

### 3 скилла (`skills/`)

Переиспользуемое алгоритмическое знание за тремя самыми ответственными агентами — точная формула вывода id, правила биннинга зон и схлопывания мега-узлов, дисциплина классификации доверия рёбер и argv-safe grep. Вызываются одноимёнными агентами, это не general-purpose инструменты.

### 3 шаблона композиций (`compositions/`)

Данные, не код — как сканированный проект превращается в зоны. `rust-cli-mcp` и `web-fullstack` — вручную настроенные догфуд-цели; `generic` — пол корректности, который всегда рисует хоть что-то, даже на репозитории, который не узнали два других шаблона.

### Контракт (`schemas/map.schema.json`)

Структурная JSON-схема, которой обязан соответствовать любой `map.json` — общая точка отсчёта для эмиттера, guardian'а и собственного клиентского валидатора forgeplan-web.

### Гейт (`scripts/map-guardian.mjs`)

Детерминированный Node-скрипт без внешних зависимостей — никогда не LLM-вызов. Шесть проверок плюс две кросс-source-проверки, независимо пересчитанные из эмитированного документа и реального состояния репозитория. Только его `exit 0` переводит карту в `"confirmed"`.

### Хук защиты записи (`hooks/`)

Fail-closed PreToolUse-гейт: ни один агент в этом пакете не может писать никуда внутри `.forgeplan/`, кроме `map/map.json` и `map/.work/**`, независимо от того, во что уболтали промпт агента. Это один из трёх независимых, слоёных контролей — см. раздел «EMITTER-safe» в ARCHITECTURE.md о том, почему одного денилиста недостаточно.

## Принципы дизайна

- **Оркестровано, не автономно в одном контексте.** Каждая стадия работает в своём изолированном контексте — та же дисциплина «генератор не проверяет сам себя», что используется в этом маркетплейсе для BMAD/SPARC/TDD.
- **Детерминировано по построению.** Content-hash id узлов, ни один узел никогда не несёт `x`/`y`, геометрией владеет чистая функция раскладки. Повторный прогон добавляет узлы, не перетасовывая уже существующие.
- **Карта производная, не авторская.** Она в .gitignore, как `lance/` — удали, перезапусти, получи то же самое (с поправкой на реальные изменения репозитория). Ни один forgeplan-артефакт этим пакетом никогда не мутируется; контроли пути записи делают это структурно, а не просто по договорённости.
- **Гейтовано честно.** Каждая гарантия этого пакета — либо реально работающая детерминированная проверка, либо явно помеченная как обеспеченная соглашением, а не структурой. Ничего не заявляется без реальной проверки за этим.
